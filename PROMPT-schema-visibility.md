## 项目背景

我在做一个面向中小学老师的 AI 知识库产品，叫"飞象 AI"。当前在做 v28 版本，理论基础是 Andrej Karpathy 的 LLM Wiki 模式（让 AI 持续编译/维护一份会复利增长的知识资产，不是每次提问都重新检索）。老师上传课件、试卷、答疑等资料 → AI 整理成 Wiki → 老师在对话里和 AI 交互，AI 会自动沉淀"老师专属规则"（schema），让 AI 越用越懂这位老师。

## 任务 A · 在对话流里加"✨ 我记住了"轻提示

### 位置

`02-personal-home.html` 的 `.chat-stream` 区域内（约第 316-438 行），具体是 `#chat-body` 里 `.chat-msg.ai` 那条 AI 回答之后，再插入一条新消息。

### 内容（文案以下完全照抄，不要改写）

```
✨ 我记住了 · 以后"含参 + 二次函数"的题，会自动归到「中考压轴 · 函数综合」分类下
```

带两个轻按钮：
- `[查看所有规则]` — 点击触发任务 B 的面板（详见下文）
- `[这条不对]` — 点击 toast 提示"（演示）已撤销这条规则"

### 视觉

- **不是普通的 AI 消息气泡**——它是一条系统级、轻量、独立的"提示行"
- 视觉上比 `.chat-msg.ai` 更轻：可以是一行带左侧 ✨ 图标的文字 + 右侧两个 ghost 按钮
- 用 `--gold-bg`（rgba(201,167,110,.10)）作为底色，左侧 2px 金色竖线作为强调
- 圆角 8px，padding 10-12px，font-size 12.5-13px
- 字体颜色用主灰，不要用纯黑；按钮用 ghost 样式（边框 `--border`，字色 `--ink-3`）

### 命名约定（重要，保持代码一致性）

新加的 class 用 `mem-` 前缀，例如：`mem-toast`、`mem-text`、`mem-actions`、`mem-btn`。这是这个项目里 schema 沉淀提示专用的命名空间。

### 参考结构（不一定照搬，看你判断）

```html
<div class="mem-toast">
  <i data-lucide="sparkles"></i>
  <span class="mem-text">我记住了 · 以后"含参 + 二次函数"的题，会自动归到「中考压轴 · 函数综合」分类下</span>
  <div class="mem-actions">
    <button class="mem-btn" onclick="openLearnedPanel()">查看所有规则</button>
    <button class="mem-btn" onclick="showToast('（演示）已撤销这条规则')">这条不对</button>
  </div>
```

## 通用约束（重要）

### 设计系统

- 严格使用 v28.css 已有的 CSS 变量：`--gold`、`--gold-bg`、`--gold-deep`、`--surface`、`--surface-2`、`--border`、`--bg` 等
- 不要写死颜色值（除非现有变量没覆盖到）
- 圆角：小控件 6-8px / 浮层 10-14px
- 图标：Lucide，stroke-width 1.75
- 字体大小遵循现有页面规则（正文 13-14px，meta 11-12px，按钮 12.5px）

### 文案规则（DESIGN-SYSTEM-v28.md 第 11 节）

- 用"飞象 AI"或"飞象"，不要写"系统""小助手""智能助手"
- 避免营销腔和自夸腔
- 解释 AI 时强调"整理""引用""追溯""记住"，避免"替老师决策"
- ✨ emoji 这一处可保留（Karpathy 原文 LLM Wiki 项目就用这个符号），其他地方不要乱加

### 代码风格

- 新加样式优先放在 02-personal-home.html 文件头部的 `<style>` 块（如果没有就新建一个）
- 不要修改 v28.css 共享文件
- JS 函数定义在文件底部 `<script>` 块或现有 v28.js 引用之前
- 保持现有缩进风格（2 空格）

### Demo 性质（关键！）

这是给老板/团队/客户看的静态 demo，不接真实后端。所有按钮交互用 `showToast()` 提示即可（toast 容器 `<div id="toast">` 已存在，函数已定义）。**唯一需要真实交互的是抽屉面板的开关**。

### 不要做的事

1. 不要给 wiki 段落加任何 hover 按钮（已讨论过，YAGNI）
2. 不要做富文本编辑器（飞象不让老师直接编辑 wiki，只通过对话）
3. 不要把 schema 显示成 JSON / markdown 配置（要显示成人话规则）
4. 不要修改任何其他 HTML 文件（01、03、04、05、06、07）
5. 不要新建独立页面（schema 不是独立页，是抽屉）
6. 不要引入任何新的 npm 包或外部库（lucide 已有）
