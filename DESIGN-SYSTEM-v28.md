# 飞象 AI 教师端 v28 · 设计系统说明

> 用于把静态 demo 转成设计可复用资产。这里记录组件用途、状态和文案规则，方便设计稿、前端实现、评审口径保持一致。

## 1. 视觉基调

- 基底：奶色纸感，强调“资料库 / 学院派 / 克制”。
- 主色：金色作为知识锚点，不作为营销高亮。
- 图标：Lucide，`stroke-width: 1.75`。
- 圆角：小控件 6-8px，浮层 10-14px。
- 阴影：轻，主要用于浮层和可点击卡片。

## 2. 设计 Token

| Token | 值 | 用途 |
|---|---|---|
| `--bg` | `#F5F3EF` | 页面背景 |
| `--surface` | `#FFFFFF` | 卡片、输入框 |
| `--surface-2` | `#FBFAF7` | 侧栏、顶栏弱背景 |
| `--surface-3` | `#F2EEE5` | hover / 轻选中 |
| `--border` | `#E8E4DC` | 默认边线 |
| `--gold` | `#A87E2C` | 主要强调 |
| `--gold-deep` | `#7A5D20` | 强文字 |
| `--gold-bg` | `rgba(201,167,110,.10)` | active 背景 |

## 3. AppShell

AppShell 由 `Sidebar`、`Topbar`、页面内容区和可选 AI 对话区组成。

状态：

- 宽侧栏：240px。
- 折叠侧栏：56px，仅保留图标。
- 对话态：隐藏 topbar，主内容切到对话页面。

## 4. Sidebar

职责：承载“去哪儿”和账号入口，不承载搜索。

结构：

```text
品牌
新对话
历史对话
分隔线
我的知识库
团队知识库
  团队空间列表
AI 题库
账号入口
```

状态：

- active：浅金背景 + 左侧 2px 金色竖线。
- hover：浅灰背景。
- collapsed：仅显示 icon，hover 显示 tooltip。
- 账号菜单：点击头像展开，包含个人信息、设置、账号与安全。

规则：

- `AI 题库` 不展示右侧数字。
- `使用指南` 不作为侧栏入口，只作为知识库内容资产存在。
- `历史对话` 紧跟 `新对话`，属于对话区。

## 5. Topbar

职责：提供当前知识上下文、全局视图切换、搜索、通知。

结构：

```text
[当前 KB] [Wiki / 知识图谱 / 文件夹] [spacer] [搜索 Wiki、文件…] [通知]
```

状态：

- `当前 KB` 始终显示。
- `ViewSwitcher` 当前视图 active。
- `搜索` 是短输入框样式触发器，不展示快捷键。
- `通知` 保持右上角轻量 icon，不收入账号菜单。

## 6. GlobalSearch

职责：跨 Wiki、文件、题目搜索。

交互：

- 点击顶栏短搜索框展开下拉搜索面板。
- `Esc` 或点外部关闭。
- `⌘K` 可唤起，但界面不展示快捷键。

结果类型：

- Wiki：知识页。
- 文件：PPT / Word / PDF 等来源文件。
- 题目：AI 题库中的结构化题目。

## 7. Wiki 首页

职责：让老师第一眼看到“AI 已经把资料织成 Wiki”。

设计要点：

- 首页不是最近文件列表。
- 新用户个人库要能看到使用指南作为内容资产。
- 团队库首页要突出学段 / 知识点浏览。

## 8. Wiki 词条页

职责：一个知识点的可消费页面。

必须包含：

- 词条标题和元信息。
- AI 整理正文。
- 来源文件引用。
- 相关题目 / 历史对话引用。
- 页面级操作：整理依据、下载、分享。

## 9. 文件夹视图

职责：满足“找原件”的路径。

规则：

- 纯列表，不做左侧文件夹树。
- 子文件夹用一行 row 表达。
- 演示阶段所有文件点击统一进入 `04-file-preview.html`。

## 10. 文件预览

职责：左原文件，右 AI 解读。

规则：

- 默认双栏 1:1。
- 不把 `data-view` 设为 `folder`，避免主内容被通用 CSS 隐藏。
- 面包屑与操作放在第二行 `entry-bcbar`，避免顶栏拥挤。

## 11. 文案规则

- 用“飞象 AI”或“飞象”，不要写“系统”“小助手”“智能助手”。
- 避免营销腔和自夸腔。
- 少用口语梗。
- 解释 AI 时强调“整理”“引用”“追溯”，避免“替老师决策”。
